# ניהול חוסרים נטו

אפליקציה מקומית לניהול חוסרים: מעלים קובץ מלאי WMS וקובץ הזמנות פתוחות בפורמט XLSX או CSV, ומקבלים חוסר נטו לכל מק"ט.

## איך מפעילים

פותחים את `index.html` בדפדפן, או עובדים דרך השרת המקומי שכבר פתוח.

אפשר להעלות את שני הקבצים יחד באזור העליון. האפליקציה מזהה מי מלאי ומי הזמנות לפי העמודות ושם הקובץ, ואפשר להחליף ביניהם אם הזיהוי לא נכון.

## עמודות מזוהות

האפליקציה מזהה כמה שמות נפוצים לעמודות:

- מק"ט: `מק"ט`, `מק'ט`, `SKU`, `מקט`
- מלאי: `מלאי מרלוג ביח'`, `מלאי`, `Quantity`
- ימי מלאי: `ימי מלאי`, `ימי מלאי 0`, `Days Inventory`
- הזמנות: `כמות בהזמנות`, `הזמנות`, `כמות`, `Quantity`
- שם מוצר: `תיאור מוצר`, `תאור מוצר`, `שם מוצר`, `Description`

בקבצי XLSX האפליקציה קוראת את הגיליון הראשון.

אפשר לערוך את עמודת `נשלח היום` ישירות בטבלה. פריטים שנשלחו נשארים מוצגים ומסומנים בירוק. אם בדוח המלאי קיימת עמודת `ימי מלאי`, פריטים עם 7 ימי מלאי ומטה יסומנו `בסיכון`. אפשר לסנן לפי חוסרים וסיכון, נשלחו, או הכל, ולהחזיר פריטים שהוסתרו דרך כפתור `פריטים בארכיון`.

דיווחי המשלוחים וההסתרות נשמרים בדפדפן המקומי. `איפוס משלוחים` מנקה רק את הדיווחים הידניים, ו-`איפוס קבצים` מנקה רק את הקבצים שהועלו.
